# VerifAI — תוכנית אב ל"מושלם" (~50 שעות)

*המטרה: מערכת שחברת ענק יכולה לבדוק, לאמת ולקנות. כל שלב מסתיים בקריטריון מדיד.*
*סדר השלבים = סדר עדיפות. כל שלב עומד בפני עצמו ונדחף ל-git בסיומו.*

---

## שלב 0 — אמת מידה: בנצ'מרק אמיתי (~8 שעות) 🎯 החשוב ביותר

בלי מספר אמיתי אין עסקה. כל השאר בא אחרי.

- [ ] **0.1** תשתית בנצ'מרק: סקריפט שמקבל תיקיית סרטונים + קובץ תוויות (AI/אמיתי), מריץ את הצינור המלא, ומפיק דוח: precision, recall, FPR, מטריצת בלבול, פירוק לפי פלטפורמה ולפי שכבת-הכרעה
- [ ] **0.2** פרוטוקול איסוף חצי-אוטומטי: רשימת 300 סרטוני יעד (150 AI / 150 אמיתי) מטיקטוק/יוטיוב/אינסטגרם, כולל מקרים קשים: קמח/קונפטי/ספריי, מצלמות אבטחה, וידאו ישן, סלואו-מושן. ההורדה דרך הטלפון של ניתאי (IP ביתי) עם ה-Share Intent הקיים
- [ ] **0.3** ריצה + דוח + תיקון הכשלים שיתגלו (לולאה: מדוד → תקן → מדוד)
- [ ] **0.4** דף "Accuracy" ציבורי באתר עם התוצאות האמיתיות

**✔ מושלם כאשר:** יש דוח על ≥200 סרטונים אמיתיים עם precision ≥90% ו-FPR ≤5%, מפורסם באתר.

## שלב 1 — המודל: מדיוק טוב לדיוק מכירתי (~12 שעות)

- [ ] **1.1** הרחבת דאטהסט ל-2,000+ דגימות: העברת `train_forever.py` לרוץ בענן (הסרת נתיבי macOS קשיחים, ריצה כ-worker ב-Railway או GitHub Actions מתוזמן)
- [ ] **1.2** איזון מקורות: לוודא שהמודל לומד "AI" ולא "מקור" — דגימות אמיתיות גם מטיקטוק/אינסטגרם (לא רק Pexels), דגימות AI גם אחרי גריסת-פלטפורמה
- [ ] **1.3** כיול הסתברויות (calibration) + בחירת ספים לפי עלות טעות: FPR נמוך חשוב יותר מ-recall מושלם למכירה
- [ ] **1.4** שדרוג מודל הפריימים: אימון על פריימים אמיתיים במקום דאטה סינתטי
- [ ] **1.5** נעילת גרסאות (requirements) ותיקון אזהרת sklearn 1.5→1.9 בטעינת המודל

**✔ מושלם כאשר:** המודל מאומן על ≥2,000 דגימות מכל הפלטפורמות, האימון רץ אוטומטית בענן כל לילה, והבנצ'מרק משלב 0 משתפר.

## שלב 2 — ערוצי קלט: לקבל כל סרטון, מכל מקום (~10 שעות)

- [ ] **2.1** **בוט וואטסאפ** (WhatsApp Business Cloud API): אותו מנוע כמו טלגרם — קבלת קובץ כמסמך = בייטים מקוריים. זה הערוץ שהקהל הישראלי חי בו
- [ ] **2.2** שרת Telegram Bot API מקומי בקונטיינר → שובר את מגבלת 20MB (עד 2GB)
- [ ] **2.3** חיזוק הורדות שרת: rotating headers, גיבוי דרך הטלפון (המכשיר כ-fetch-agent כשה-CDN חוסם), עדכון דפוסי `aigcLabelType` של טיקטוק לשדות 2026
- [ ] **2.4** תור עבודות (queue) לניתוחים כבדים במקום חסימת request — יציבות תחת עומס

**✔ מושלם כאשר:** אפשר לשלוח סרטון מכל אפליקציה מרכזית (טלגרם/וואטסאפ/Share בטלפון/אתר/API) ולקבל תשובה תוך <30 שניות, כולל קבצים גדולים.

---

## שלב 3 — מוצר ועיצוב: להיראות כמו מוצר של מאות מיליונים (~10 שעות)

- [ ] **3.1** **דף נחיתה חדש**: מיצוב ברור ("שכבת האמת של הווידאו"), דמו חי — המבקר מדביק קישור טיקטוק ורואה ניתוח, לוגו-בר של שכבות הזיהוי, מספרי הבנצ'מרק
- [ ] **3.2** **דוח ניתוח ויזואלי**: במקום "AI 93%" — דוח מלא: איזו שכבה הכריעה, ציר זמן פריימים חשודים, ממצאי C2PA, תווית פלטפורמה. זה ההבדל בין גאדג'ט לכלי forensics
- [ ] **3.3** דשבורד B2B: גרף שימוש, היסטוריית בדיקות, ניהול מפתחות, החלפת תוכנית — ברמת גימור של Stripe
- [ ] **3.4** מובייל: ליטוש UX של תוצאת הבדיקה (אותו דוח ויזואלי), אונבורדינג קצר, עמידה בדרישות חנויות
- [ ] **3.5** עקביות מותג: שם, לוגו, צבעים, דומיין אמיתי (לא vercel.app) — verifai.app או דומה

**✔ מושלם כאשר:** אדם זר שנוחת באתר מבין תוך 10 שניות מה זה, מנסה דמו תוך 30 שניות, ומקבל דוח שנראה כמו מוצר אבטחה רציני.

## שלב 4 — Enterprise readiness: מה שחברת ענק בודקת לפני צ'ק (~8 שעות)

- [ ] **4.1** תיעוד API מלא: OpenAPI מלוטש, מדריכי אינטגרציה, SDK פייתון/JS מפורסמים ל-PyPI/npm
- [ ] **4.2** תפעול: לוגים מובנים, ניטור שגיאות (Sentry), health checks, alerting, יעד uptime
- [ ] **4.3** אבטחה: סיבוב מפתחות, הגבלת קצב לפי מפתח, מחיקת סרטונים מיידית אחרי ניתוח (privacy by design), עמוד מדיניות פרטיות
- [ ] **4.4** טסטים: כיסוי הצינור המרכזי (unit + integration), CI שרץ על כל push
- [ ] **4.5** דוח JSON מפורט ב-API (explainability) — פירוק ההכרעה לשכבות, לצרכי audit של הלקוח

**✔ מושלם כאשר:** מהנדס של חברת ענק יכול לעשות אינטגרציה לבד תוך שעה, וה-due diligence הטכני לא מוצא חורים.

## שלב 5 — החפיר הטכנולוגי: להקדים את השוק (~4 שעות + מתמשך)

- [ ] **5.1** הרשמה ל-SynthID Detector waitlist + אינטגרציה כשתיפתח (שכבת הזיהוי החזקה בעולם, חינם)
- [ ] **5.2** ניטור הטמעת C2PA של טיקטוק בהורדות — כשיגיע לכל הסרטונים, זו הכרעה קריפטוגרפית כמעט לכל סרטון
- [ ] **5.3** בדיקת זיהוי סימני מים פתוחים (Meta Video Seal וכד')
- [ ] **5.4** לולאת למידה מתמשכת: כל בדיקת משתמש (בהסכמה) → דגימת אימון → המודל משתפר כל יום

**✔ מושלם כאשר:** כל טכנולוגיית provenance חדשה שנפתחת — VerifAI תומכת בה תוך שבוע.
